üöÄ Quick Start Guide

Get started with the Cloud Monitoring API in 3 simple steps:

1 Start the Backend: Run python main.py in the backend directory
2 Test Connection: Visit http://localhost:8000/health to check if everything works
3 Start Monitoring: Use the endpoints below to interact with the system

üì° API Endpoints

GET /health
Check the overall system health including AI status and active connections.
GET /api/metrics
Get current system metrics (CPU, Memory, Latency, Error Rate) with timestamp.
GET /api/incidents
List all available incident types (CPU Spike, Memory Leak, API Failure, Network Latency).
POST /api/incidents/trigger
Trigger a single incident for testing. Send JSON with incident_id.
POST /api/incidents/trigger Content-Type: application/json { "incident_id": "cpu_spike" }
POST /api/incidents/trigger-multiple
Trigger multiple incidents simultaneously. Send JSON with array of incident_ids.
POST /api/incidents/trigger-multiple Content-Type: application/json { "incident_ids": ["cpu_spike", "memory_leak", "api_failure"] }
GET /api/ai/analyze
Request manual AI analysis of current system state. Requires Ollama + Llama 3.2 1B.

üîÑ WebSocket Connection

WS ws://localhost:8000/ws
Real-time connection for live metrics updates and incident notifications.

WebSocket Messages:

// Connect to WebSocket const ws = new WebSocket('ws://localhost:8000/ws'); // Send ping to keep connection alive ws.send(JSON.stringify({ "type": "ping" })); // Receive real-time updates ws.onmessage = function(event) { const data = JSON.parse(event.data); console.log('Received:', data); };
